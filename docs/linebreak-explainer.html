2. Penraft 现在的特殊行为

Penraft 为了让 WYSIWYG 模式下「按一次 Enter 真的换一行」更直觉，做了一个偷偷的处理：

自动加硬换行（applyHardBreaks）
当你在 Milkdown（渲染模式）里只按一次 Enter，Penraft 会自动在行尾追加两个空格 ··，把它升级成硬换行。
所以单 Enter 在 Penraft 里 = 硬换行（紧贴），而不是标准 Markdown 的「忽略 / 接成一行」。

5. 我的建议

我不建议把视觉间距完全统一，原因有三：
  1. 段落 vs 换行 是 Markdown 的真实语义，不是可有可无的样式。导出到 Typora、Obsidian、GitHub 时，那里就是用这两种间距来表达「这是新段」还是「这是同段续写」的。Penraft 强行抹平，到了别的工具又会显示出来。
  2. 抹平成紧贴（方案 A）= 阅读体验差，长文里完全分不清段落边界。
  3. 抹平成宽间距 = 把硬换行 <br> 变没用，因为它和段落看起来一样了。

真正值得讨论的一个改动

Penraft 那个自动加硬换行的行为（applyHardBreaks），其实有点偏离 Markdown 标准。它让你「单按 Enter」变成 <br>（紧贴），但标准 Markdown 里这种写法应该被忽略或当成空格。

如果关掉这个自动行为，那么在 Milkdown 里：

  • 按 Enter 一次 = 新段落（Milkdown 默认行为）
  • 所有 Enter 都产生一致的 16px 段落间距
  • 「间距」自然就统一了，并且 Markdown 是干净的标准格式
  • 代价：你之前以为是「同段续写」的地方，会被分成两个段

你需要做的决定

  1. 保持现状（推荐 → 语义清晰，跨工具一致）
  2. 关掉 applyHardBreaks，让所有 Enter 都变段落，间距自然统一
  3. 强行用 CSS 抹平间距（A / B / C 之一），不改 Markdown 行为
